Build a FULL “Career Intelligence + Apply Journey” system inside my existing app, but keep the UX clean, minimal, and NOT like a job board.

NORTH STAR (User Outcome)
A user enters, the system understands their profile, recommends 3–5 realistic career paths, curates a small set of high-fit jobs ONLY after a path is chosen, then guides them to tailor a resume (word-like editor) and leave with an application-ready pack (PDF/DOCX, optional cover letter). Intelligence exists in the same place, but it must not block applying.

IMPORTANT PRODUCT RULES (no overlap, minimal)
- Intelligence drives actions; jobs are downstream of intelligence.
- Never show “all jobs in the market” as the default.
- Keep the critical path fast: Upload → Path → Job → Tailor → Apply.
- Use at most ONE primary score per screen. Prefer High/Med/Low labels over %.
- All AI outputs must be grounded; if not grounded, mark as “Needs confirmation.”

────────────────────────────────────────
0) ENTRY LAYER – USER ONBOARDING & INPUT
────────────────────────────────────────

Goal: understand who the user is and what they want, with minimal friction.

User inputs supported:
1) Upload resume (PDF/DOCX)
2) Paste resume text
3) Paste LinkedIn URL
4) Paste portfolio website URL
5) Type a short intent prompt (optional): “I want Legal Ops roles in NYC”
6) Voice input (optional, via mic button; transcribe to text)

UI rule:
- Default CTA is “Upload resume”
- Search/intent input is secondary (small field)
- Voice is optional (mic icon next to text field), not primary.

System action:
- Parse and normalize into structured resume/profile JSON
- Extract:
  - career stage (student / <1 yr / 1–5 / 5+)
  - practice area (litigation/corporate/IP/etc.)
  - years of experience (estimate)
  - skills (legal + tech)
  - education
  - inferred intent (if provided) + constraints (location/remote/seniority)

Output to user:
- “Your current position:” (1 concise card)
  - Example: “Mid-Level Litigation Associate (5 years)”
  - “Strengths: Research, Drafting”
  - “Gaps: Limited tech exposure”
Keep this tight and honest.

────────────────────────────────────────
1) POSITION IDENTIFICATION ENGINE
────────────────────────────────────────

Goal: identify where the user stands (clarity boost).

Implement a server-side module:
- server/lib/intelligence/position-engine.ts

It takes normalized profile JSON and returns:
- careerStage label
- experienceCluster label (e.g., “5 years litigation”)
- strengths (max 3)
- limitations/gaps (max 3)

Must be grounded in resume evidence; if uncertain, say “Not enough evidence.”

────────────────────────────────────────
2) CAREER MAP GENERATION (3–5 PATHS MAX)
────────────────────────────────────────

Goal: show upward/lateral paths without overwhelming.

Career Map UI (cards only):
For each path (max 3–5):
- Role name
- Fit label: High / Medium / Low (NOT %)
- “Why this fits” (2–3 bullets, grounded)
- “What you’ll need” (2–3 bullets, gaps)
- Optional “Reality check” (1 short line if it’s a stretch)
- CTA: “Choose this path”

No job list on this screen.

Implement:
- server/lib/intelligence/career-map-engine.ts

Output schema:
paths: [
  {
    pathId,
    roleName,
    fitLabel,               // High/Medium/Low
    whyFit: [{ text, grounded, evidenceRefs[] }],
    gaps: [{ text }],
    realityCheck?: string,
    nextStep?: string
  }
]

Evidence refs must point to resume JSON stable IDs.

────────────────────────────────────────
3) JOB CURATION ENGINE (LIVE ROLES, PATH-GATED)
────────────────────────────────────────

Goal: curate a small list of jobs once the user chooses a path.

Rules:
- Jobs are shown ONLY after path selection.
- Default list: top 10 jobs (max 20) to avoid job-board feel.
- Each job card shows:
  - Title, Company
  - 3 key requirements (max)
  - “Fit label” High/Med/Low (NOT % by default)
  - CTA: “Apply with Model Resume”
  - Secondary: “Save”

Optional “Explore more” is behind a click (not default).

Implement:
- server/lib/jobs/job-curation-engine.ts

Ranking logic:
- must consider selectedPath role taxonomy tags
- skills overlap
- seniority alignment
- location/remote constraints (if provided)
Return:
jobs: [
  { jobId, title, company, location, remote, keyReqs[], fitLabel, whyRecommendedLine }
]

10) API ROUTES (END-TO-END)
────────────────────────────────────────

A) Intake
POST /api/intake
Body: { uploadId | pastedText | linkedinUrl | portfolioUrl | intentText }
Returns: { resumeId, profileSnapshot, positionCard, toConfirmItems[] }

B) Intelligence Home
GET /api/intelligence/home?resumeId=...
Returns: { positionCard, careerMapPaths, selectedPathId?, jobsPreview(<=3) }

C) Career Map
POST /api/career-map/generate { resumeId }
POST /api/career-map/select { resumeId, pathId }

D) Curated Jobs (path-gated)
GET /api/jobs/curated?resumeId=...&pathId=...
Optional:
GET /api/jobs/explore?resumeId=...&pathId=...&query=...  // not default

E) Compare
POST /api/jobs/compare { resumeId, jobIds[] }  // max 3

F) Resume Editor
GET /api/resume/:resumeId/editor?jobId=...&mode=...
POST /api/resume/:resumeId/editor/save
GET /api/resume/:resumeId/export/pdf?jobId=...&mode=...
GET /api/resume/:resumeId/export/docx?jobId=...&mode=...
GET /api/resume/:resumeId/export/apply-pack?jobId=...&mode=...

G) Growth Loop
GET /api/growth-loop?resumeId=...&pathId=...

H) Health
GET /api/health
