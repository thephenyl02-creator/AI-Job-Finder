FEATURE 1: CAREER DIAGNOSTIC REPORT (the core “10/10” value)
========================================================
Add a new route/page: /diagnostic

Trigger:
- After resume upload OR user clicks “Run Career Diagnostic”

Backend:
- Create endpoint POST /api/diagnostic/run
- Input: resumeId, optional targetPath (if user selects one)
- Output: a structured DiagnosticReport JSON (store in DB for caching)

DiagnosticReport must include:
A) Top 3 recommended career paths (with confidence scores)
B) Role Readiness Ladder:
   - READY (roles user can apply now)
   - NEAR-READY (roles needing 2–6 weeks of targeted work)
   - STRETCH (roles needing 3–6 months)
   For each role: include “why” + top 3 blockers + top 3 strengths.
C) Skill Cluster Scores (0–100) with explainability:
   - Legal Domain Expertise
   - Legal Ops / Process
   - Product Thinking
   - Data / Analytics
   - AI Literacy (LLMs, RAG, evaluation)
   - Technical Tools (SQL/Python/automation)
   - Stakeholder / Leadership
   For each cluster: list evidence lines from the resume (quotes/snippets) and missing signals.
D) 30-Day Transition Plan:
   - Week 1–4 actions with time estimate (hrs), outputs (deliverables), and measurable goals.
   - Each task must map to (i) skill gaps and (ii) real job requirements from our jobs DB.
E) “Brutal Honesty” section:
   - 3 direct statements about what is holding the user back most (no sugarcoating).
F) Recommended next actions:
   - “Tailor resume for top match”
   - “Generate LinkedIn headline + About section”
   - “Apply to 5 roles I’m ready for”

DB:
- Add table diagnostic_reports:
  - id, userId, resumeId, createdAt
  - reportJson (jsonb)
  - topPaths, readinessSummary, overallReadinessScore
Cache per (userId,resumeId) and regenerate only if resume changes.

Scoring:
- Implement deterministic, explainable scoring:
  - Extract requirements from job descriptions into clusters + keywords
  - Extract resume skills/experience into clusters + evidence
  - Compute cluster overlap, seniority alignment, domain alignment
  - Create readiness tiers based on thresholds + blockers
- Do NOT output “magic numbers” without evidence. Every score must have “why”.
