# Fix: Job descriptions render differently between admin and user views

## Problem

Jobs that have `structuredDescription` populated render beautifully with icons and labeled sections (via `StructuredDescriptionView`). Jobs without it fall back to raw text parsing (via `DescriptionContent`) which looks messy and unformatted. The autopilot pipeline in `scheduled-scraper.ts` never calls `extractStructuredDescription()` during Phase 4 (AI Categorization), so scraped jobs never get their `structuredDescription` field populated. Only manually backfilled jobs look good.

## Root Cause

In `server/lib/scheduled-scraper.ts`, the Phase 4 categorization loop calls `categorizeJob()` and saves fields like `roleCategory`, `keySkills`, `aiSummary`, etc. — but it never calls `extractStructuredDescription()` from `server/lib/description-extractor.ts`. The `structuredDescription` JSONB column stays null for all autopilot-scraped jobs.

The rendering logic in `client/src/pages/job-detail.tsx` (around line 1023) checks:
```ts
const structured = parseStructuredDescription(job.structuredDescription);
if (structured) {
  return <StructuredDescriptionView data={structured} />;  // clean view
}
// Falls back to raw DescriptionContent — messy regex-based parsing
```

## Required Changes

### 1. `server/lib/scheduled-scraper.ts`

**Add import** at the top with the other imports:
```ts
import { extractStructuredDescription } from './description-extractor';
```

**In Phase 4** (the categorization loop, inside the `batch.map(async (job) => { ... })` block), after the `updateData` object is created and BEFORE the `if (result.reviewStatus === "rejected")` check, add:

```ts
// Extract structured description for consistent rendering
if (job.description && !job.structuredDescription) {
  try {
    const structured = await extractStructuredDescription(job.description, job.company, job.title);
    if (structured) {
      updateData.structuredDescription = structured;
    }
  } catch (extractErr: any) {
    logWarn('AI', `Structured extraction failed for job ${job.id}: ${extractErr.message?.slice(0, 80)}`);
  }
}
```

**Add a new Phase 4.6** after the existing Phase 4.5 (Description Formatting) and before Phase 5 (Alert matching). This backfills structured descriptions for already-categorized jobs that are missing them:

```ts
// === PHASE 4.6: Structured description backfill for existing jobs ===
logInfo('PHASE', '--- Phase 4.6: Structured Description Backfill ---');
try {
  const allActiveForStructured = await storage.getActiveJobs();
  const needsStructured = allActiveForStructured.filter(j =>
    j.description && j.description.length > 100 && !j.structuredDescription
  );

  if (needsStructured.length > 0) {
    logInfo('AI', `Extracting structured descriptions for ${needsStructured.length} jobs...`);
    let structuredCount = 0;
    const structBatchSize = 3;
    for (let i = 0; i < needsStructured.length; i += structBatchSize) {
      const batch = needsStructured.slice(i, i + structBatchSize);
      await Promise.all(
        batch.map(async (job) => {
          try {
            const structured = await extractStructuredDescription(job.description, job.company, job.title);
            if (structured) {
              await storage.updateJob(job.id, { structuredDescription: structured } as any);
              structuredCount++;
            }
          } catch (err: any) {
            logWarn('AI', `Structured extraction failed for job ${job.id}: ${err.message?.slice(0, 80)}`);
          }
        })
      );
      if (i + structBatchSize < needsStructured.length) {
        await new Promise((r) => setTimeout(r, 500));
      }
    }
    logSuccess('AI', `Extracted structured descriptions for ${structuredCount}/${needsStructured.length} jobs`);
  } else {
    logInfo('AI', 'All jobs already have structured descriptions');
  }
} catch (err: any) {
  logError('AI', 'Structured description backfill failed', { error: err.message });
  errors.push(`Structured backfill: ${err.message}`);
}
```

### 2. After deploying, trigger a scrape run

Go to `/admin/scraper` and click "Run Now". Phase 4.6 will automatically backfill all existing jobs with structured descriptions on the first run. No manual backfill endpoint needed after this.

## What NOT to change

- `job-detail.tsx` — rendering logic is correct, it just needs the data
- `description-extractor.ts` — already works perfectly
- `structured-description-view.tsx` — already works perfectly
- `structured-description.ts` (parseStructuredDescription) — already works perfectly
- No admin vs user permission issue exists — it's purely a data completeness problem
