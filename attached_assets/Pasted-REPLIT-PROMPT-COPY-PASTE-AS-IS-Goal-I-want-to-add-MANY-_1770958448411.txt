1) DATA MODEL: ADD INGESTION + QA FIELDS TO JOB
-------------------------------------------------------
Extend the canonical NormalizedJob schema to include:

type QAStatus = "failed" | "passed" | "needs_review";
type IngestStatus = "imported" | "normalized" | "qa_passed" | "published" | "rejected";

type NormalizedJob = {
  id: string;
  title: string;
  company: string;
  location: string;
  remoteType: "onsite" | "hybrid" | "remote" | "unknown";
  roleCategory: string;
  seniority: string;

  summary: string;
  minimumQualifications: string[];
  preferredQualifications: string[];
  coreSkills: string[];
  responsibilities: string[];

  compensation: string;
  originalUrl: string;
  originalDescription: string;

  // new fields for scale + checks
  source: string;                 // "LinkedIn", "Greenhouse", "Lever", "Manual", etc.
  sourceId: string;               // external id if available
  jobHash: string;                // hash(title+company+location+originalUrl) for dedupe
  postedAt: string | null;
  expiresAt: string | null;

  lawyerFirstScore: number;       // 0-100 (how relevant to lawyers/legal tech)
  excludeReason: string | null;   // e.g. "Engineering-only role"

  ingestStatus: IngestStatus;
  qaStatus: QAStatus;

  qaErrors: { code: string; message: string }[];  // why it failed
  qaWarnings: { code: string; message: string }[];// allowed but flagged
  status: "draft" | "published";  // published only when qa_passed + admin publish

  createdAt: string;
  updatedAt: string;
};

Rules:
- status="published" only if ingestStatus="published" AND qaStatus="passed".
- Users query ONLY published jobs.

4) QA CHECKS (THIS IS WHAT PREVENTS ADMIN/USER MISMATCH)
-------------------------------------------------------
Implement a QA function that returns:
{ qaStatus, errors[], warnings[], lawyerFirstScore, excludeReason }

PUBLISH-BLOCKING ERRORS (fail QA):
E_TITLE_MISSING: title < 3 chars
E_COMPANY_MISSING: company < 2 chars
E_SUMMARY_SHORT: summary < 30 chars
E_SKILLS_TOO_FEW: coreSkills < 3
E_MINQ_EMPTY: minimumQualifications empty (or responsibilities empty) -> choose one as required
E_ROLECATEGORY_EMPTY: roleCategory empty
E_ENGINEERING_ONLY: detected engineering-only job (see rules below)
E_DUPLICATE: jobHash already exists and is active

WARNINGS (allowed but flagged):
W_LOCATION_UNKNOWN
W_PREFERRED_EMPTY
W_RESP_TOO_FEW (responsibilities < 2)
W_LOW_LAWYER_SCORE (lawyerFirstScore < threshold)
W_TOO_MUCH_RAW (originalDescription huge / untrimmed)

DEFAULT THRESHOLDS:
- lawyerFirstScore must be >= 60 to auto-pass without review
- 40–59 => needs_review
- < 40 => fail (unless admin overrides with “Include anyway” toggle)

6) DEDUPLICATION (AVOID MULTIPLE COPIES)
-------------------------------------------------------
Compute jobHash = sha256(lower(title|company|location|originalUrl))
Before insert/publish:
- If same hash exists and not expired/rejected -> mark as duplicate and skip or update existing.

Also handle “same job posted with slight changes”:
- fallback hash without URL: sha256(lower(title|company|location))
